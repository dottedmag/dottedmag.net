DE/compositor development notes - common DRM device operatins

What can compositor do with DRM devices once it has found them?

These devices respond to a number of ioctls. All DRM devices respond to a set of common ioctls that cover display controller functionality. There are also driver-specific ioctls used chiefly by Mesa for rendering operations.

DRM master:

  • DRM_IOCTL_GET_MAGIC

  • DRM_IOCTL_MAP_BUFS ??
  • DRM_IOCTL_FREE_BUFS ??
  • DRM_IOCTL_SET_SAREA_CTX ??
  • DRM_IOCTL_GET_SAREA_CTX ??

Various:

  • DRM_IOCTL_VERSION returns some (useless) metadata about driver.

Obsolete:

  • DRM_IOCTL_GET_UNIQUE returns some ID for card. Long-time unused except to avoid libdrm breakage.
  • DRM_IOCTL_IRQ_BUSID returns IRQ number of of device with given PCI bus ID.
