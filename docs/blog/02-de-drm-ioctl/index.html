DE/compositor development notes - common DRM device operatins

What can compositor do with DRM devices once it has found them?

These devices respond to a number of ioctls. All DRM devices respond to a set of common ioctls that cover display controller functionality and there are driver-specific ioctls used chiefly by Mesa for rendering operations.

Version and capabilities

  • DRM_IOCTL_GET_CAP
  • DRM_IOCTL_SET_CLIENT_CAP
  • DRM_IOCTL_SET_VERSION

DRM master

  • DRM_IOCTL_GET_MAGIC
  • DRM_IOCTL_AUTH_MAGIC
  • DRM_IOCTL_SET_MASTER
  • DRM_IOCTL_DROP_MASTER
  • DRM_IOCTL_MODE_CREATE_LEASE
  • DRM_IOCTL_MODE_GET_LEASE
  • DRM_IOCTL_MODE_LIST_LESSEES
  • DRM_IOCTL_MODE_REVOKE_LEASE

Framebuffer handling

  • DRM_IOCTL_MODE_ADDFB2
  • DRM_IOCTL_MODE_ADDFB
  • DRM_IOCTL_MODE_GETFB2
  • DRM_IOCTL_MODE_GETFB
  • DRM_IOCTL_MODE_RMFB
  • DRM_IOCTL_MODE_DIRTYFB

Display control

  • DRM_IOCTL_CRTC_GET_SEQUENCE
  • DRM_IOCTL_CRTC_QUEUE_SEQUENCE
  • DRM_IOCTL_WAIT_VBLANK
  • DRM_IOCTL_MODE_PAGE_FLIP

Memory management

  • DRM_IOCTL_MODE_CREATE_DUMB
  • DRM_IOCTL_MODE_DESTROY_DUMB
  • DRM_IOCTL_MODE_MAP_DUMB
  • DRM_IOCTL_GEM_CLOSE
  • DRM_IOCTL_GEM_FLINK
  • DRM_IOCTL_GEM_OPEN

  • DRM_IOCTL_MODE_GETPROPERTY

  • DRM_IOCTL_MODE_OBJ_SETPROPERTY

  • DRM_IOCTL_MODE_GETCRTC

  • DRM_IOCTL_MODE_GETPLANERESOURCES

  • DRM_IOCTL_MODE_SETCRTC
  • DRM_IOCTL_MODE_SETGAMMA

Modesetting

  • DRM_IOCTL_MODESET_CTL
  • DRM_IOCTL_MODE_ATOMIC

Synchronization
